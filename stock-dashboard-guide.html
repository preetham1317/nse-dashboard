Your personal NSE stock intelligence dashboard

A complete, no-surprises plan before a single line of code gets written: what gets built, in what order, what it costs, what's free, and exactly how it will look and feel on web and mobile.

The whole system answers one question: "Out of everything I track, what deserves my attention today — and why?"
Build Plan

Four phases, built in order

Version 1 ships first and is fully usable on its own. Each later phase only starts once the one before it is live and trusted — same discipline as the bot.

01

Foundation Build first

2–3 weeks
  • Your ~120–150 stock universe, organized by sector
  • Daily price/volume/delivery-% collection via Angel One SmartAPI
  • Technical indicators: EMA 20/50/200, RSI, MACD, ATR, VWAP, ADX, Bollinger
  • Portfolio page — your actual holdings, live-tracked
02

Ranking & Alerts

2–3 weeks
  • Sector strength — sector inherits health of its member stocks
  • Flag-based ranking (not a composite score — see below)
  • Morning and evening Telegram briefings
03

AI Explanation Layer

3–4 weeks
  • Claude Haiku explains why a stock ranked where it did, in plain language
  • Lightweight news headline summaries (context, not signal)
  • "Why did X fall?" / "Why is Y ranked #1?" query support
04

Optional — later Only if needed

Not scoped yet
  • Backtesting flag combinations against actual outcomes
  • Historical flag-count charts per stock
  • Custom flag weighting once you have real data on what matters
Fundamentals data (PE, ROE, promoter holding, etc.) isn't covered by Angel One and updates quarterly — this is resolved with a fully free stack: yfinance for ratios/statements and NSE's own filings (via nsepython/nselib) for shareholding data. No paid subscription needed. Full detail in the cost table below.

Build brief

Finalized decisions from planning. If work on this project resumes from this file alone, build against the spec below rather than re-deriving it.

ScopePersonal use only — one user, no distribution, no SEBI research-analyst exposure. Not a friends-facing tool.
Build orderPhase 1 (Foundation) first and fully usable standalone. Do not start Phase 2 until Phase 1 is running and trusted. Do not start this project's Phase 1 until bot_v26 paper-trading has produced a clean dataset — these are separate systems, but only one gets active build attention at a time.
Ranking methodFlag count, never a composite score. Rank by number of bullish conditions matched (e.g. 6/8), tie-break on rarer/higher-signal flags (sector strength > a single EMA cross). No weighted 0–100 number anywhere in the system.
Market data sourceAngel One SmartAPI — reuse the existing bot's account and auth (getMarketData bulk endpoint, not ltpData for options; for equities ltp/OHLCV is fine).
Fundamentals sourceyfinance (PE, EPS, ROE, margins, market cap, raw statements — ROCE and multi-year growth computed from statements, not pulled pre-built) + nsepython/nselib/jugaad_data (promoter/FII/DII shareholding, quarterly filings, direct from NSE). No Screener.in dependency, no paid data feed.
AI roleClaude Haiku explains flags already computed in code — it never invents a signal, score, or verdict. Cost-optimized, same model tier as the bot's scan calls.
DeliveryResponsive single web dashboard (mobile + desktop from one build, no native app) + Telegram morning/evening briefings, reusing the bot's existing Telegram bot infrastructure where possible.
HostingDefault: existing PythonAnywhere account. Flag if this needs its own always-on slot alongside the bot — Vultr Mumbai VPS is the fallback, not the default.
Visual styleLight background only, no dark mode anywhere. Glassmorphism: translucent white cards, backdrop blur, soft shadows, cool light-blue/mint gradient background. Teal = bullish/positive, amber = caution, rose = bearish — never used for anything but flag/status color, not decoration.
Logging disciplineCarry over the bot's hard-won principle: every skip/rejection/fallback path must emit an explicit log line. No silent failures, no fake fallback values standing in for missing data.
The One Change From ChatGPT's Plan

Flags, not a composite score

A single number like "95" hides its own reasoning and implies precision that isn't there. Flags keep every ranking explainable at a glance — and match what you'd already decided on for this project.

Composite score (not doing this)

95
Fundamental +24, Technical +23, Sector +15 … why these weights?

Flag count (what we're building)

6 / 8
✓ Above EMA20 ✓ Sector strong ✓ Volume 1.8× MACD cross
